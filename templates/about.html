Problem Statement

Media companies face the daily challenge of scheduling advertisements across television channels or digital platforms. Since broadcasting time is a strictly limited resource, and each advertisement varies in duration, budget value, priority, and preferred slot, selecting the optimal combination is an NP-hard optimization problem.

This system applies standard algorithms to maximize total revenue. The table below summarizes the theoretical characteristics of each algorithm:

Strategy Time Complexity Optimality
Greedy O(N log N) Heuristic
0/1 Knapsack O(N × W) Guaranteed
WIS (Interval) O(N log N) Guaranteed

Greedy Scheduling Strategy

The Greedy Scheduling strategy uses a heuristic approach to arrange ads. It sorts the advertisement pool by multiple criteria in descending importance and schedules them sequentially:

  1. Highest Priority First: High priority campaigns are given precedence.
  2. Highest Budget Second: Ties in priority are broken by selecting the campaign yielding higher revenue.
  3. Smallest Duration Third: Remaining ties are resolved by picking the shortest ad, leaving space for other campaigns.
Strengths: Extremely fast execution speed. Practical for massive datasets containing millions of campaigns.
Weaknesses: Does not guarantee the maximum possible revenue. Can leave significant slot time empty.

0/1 Knapsack Dynamic Programming

By mapping the ad allocation challenge to the classical 0/1 Knapsack problem, the system determines the absolute mathematical maximum budget combination. The slot's broadcasting capacity (e.g. 180 minutes) represents the weight capacity, the advertisement durations represent item weights, and the budgets represent values.

The recurrence relation solved by the dynamic programming solver is:

dp[i][w] = max( dp[i-1][w], budget[i] + dp[i-1][w - duration[i]] )
Strengths: Mathematically guarantees the absolute highest possible revenue allocation.
Weaknesses: Requires O(N × W) memory and runtime, which scales with slot capacity size and number of campaigns.

Weighted Interval Scheduling (Bonus)

Unlike the Knapsack problem, which assumes ads can be shifted anywhere inside a slot, Weighted Interval Scheduling handles advertisements that specify strict start and end times. It finds the subset of mutually compatible (non-overlapping) intervals that yields the maximum possible total budget.

The algorithm sorts intervals by end time $f_i$ and uses binary search to compute the predecessor array $p(j)$, which represents the latest compatible interval before $j$. The DP recurrence relation is:

M[j] = max( M[j-1], budget[j] + M[p(j)] )
Strengths: Optimizes fixed time intervals and prevents overlap conflicts. Runs in highly efficient O(N log N) time.
Weaknesses: Restricts flexibility since ads cannot shift times to fill empty capacity gaps.
